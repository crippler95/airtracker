# Contributing to AirTracker

Thanks for your interest in improving AirTracker! This project is a native macOS menu-bar app that streams AirPods head-orientation as a head tracker for games. Contributions of all kinds are welcome.

## Building

AirTracker is a Swift package. To build:

```sh
swift build
```

To build and run the menu-bar app bundle:

```sh
make run
```

## Running tests

```sh
swift test
```

Please make sure the suite is green before opening a pull request.

## Project layout

| Path | Description |
|------|-------------|
| `Sources/AirTrackerCore` | Platform-independent core library: quaternion math, the orientation pipeline, UDP / WebSocket / HTTP transports, and the CLI. |
| `Sources/AirTracker` | The SwiftUI menu-bar UI executable. |
| `Tests/AirTrackerCoreTests` | Unit tests for the core library. |

Keeping platform-independent logic in `AirTrackerCore` makes it testable without a UI and keeps the SwiftUI layer thin.

## Coding style

- Match the style of the surrounding code.
- Avoid gratuitous comments — let clear names and small functions carry the intent.
- Prefer adding logic to `AirTrackerCore` (with tests) over the UI layer where possible.

## Reporting issues

When filing a bug, please include a diagnostics bundle. It is generated by:

```sh
airtracker diagnostics
```

The bundle is redacted and safe to share. It captures the information needed to reproduce most issues.

## Pull requests

- Keep tests green (`swift test`).
- Add or update tests for behavior changes.
- Keep changes focused and describe them clearly in the PR.
